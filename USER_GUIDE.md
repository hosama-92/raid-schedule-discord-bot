# 레이드 스케줄 봇 사용 설명서

공대원들이 가능한 날짜와 시간을 입력하면, 모두 함께 레이드를 진행할 수 있는 날짜를 자동으로 찾아주는 봇입니다.

## 공대원 사용법

### 1. 가능한 일정 입력하기

공대장이 올린 스케줄 메시지에서 `주간 일정 입력하기` 버튼을 누릅니다.

화면에 표시된 날짜 중 참여 가능한 날짜를 누르고, 그날 몇 시 이후부터 가능한지 입력합니다.

```text
21       → 오후 9시 이후 가능
21:30    → 오후 9시 30분 이후 가능
오후 9시 → 오후 9시 이후 가능
```

시간을 입력하지 않고 제출하면 `아무때나 가능`으로 저장됩니다.

참여할 수 없는 날짜는 선택하지 않으면 됩니다.

### 2. 모든 날짜에 같은 시간 입력하기

해당 기간의 모든 날짜에 참여할 수 있다면 `전부 가능` 버튼을 누릅니다.

한 번 입력한 시간이 모든 날짜에 동일하게 적용됩니다.

### 3. 입력한 일정 수정하기

수정할 날짜 버튼을 다시 누르고 새로운 시간을 입력하면 기존 내용이 변경됩니다.

해당 날짜의 입력을 지우려면 입력창에 다음 중 하나를 입력합니다.

```text
삭제
없음
```

내가 입력한 모든 날짜를 한꺼번에 지우려면 `내 일정 전체 삭제` 버튼을 누릅니다.

### 4. 결과 확인하기

누군가 일정을 입력하거나 수정하면 달력 이미지가 자동으로 갱신됩니다.

달력에서는 다음 내용을 확인할 수 있습니다.

- 날짜별 입력 인원
- 공대원별 가능 시간
- 6명 이상 함께 가능한 날짜
- 함께 시작할 수 있는 가장 빠른 시간
- 아직 일정을 입력하지 않은 사람의 서버 별명

예를 들어 6명이 각각 오후 8시~9시 이후 가능하다면, 모두 가능한 시간인 오후 9시 이후로 표시됩니다.

## 공대장 사용법

### 1. 주차를 선택하여 스케줄 만들기

`/스케줄생성` 명령어를 입력하고 제목과 기간을 선택합니다.

```text
/스케줄생성 제목: 침식레이드 기간: 이번 주
```

선택 가능한 기간은 다음과 같습니다.

- 이번 주
- 다음 주
- 다다음 주

각 주차는 수요일부터 다음 주 화요일까지입니다.

### 2. 원하는 날짜 범위로 만들기

수요일 기준이 아닌 별도의 기간이 필요하면 `/스케줄수동생성`을 사용합니다.

```text
/스케줄수동생성 제목: 침식레이드 시작일: 2026-06-26 종료일: 2026-07-02
```

날짜는 `YYYY-MM-DD` 형식으로 입력하며, 최대 14일까지 지정할 수 있습니다.

### 3. 스케줄 목록 확인하기

```text
/스케줄목록
```

현재 입력을 받고 있거나 확정된 스케줄을 확인할 수 있습니다.

목록에서는 참여 인원과 아직 입력하지 않은 사람의 서버 별명도 함께 확인할 수 있습니다.

### 4. 레이드 일정 확정하기

`/스케줄목록`에서 `확정: 스케줄 이름` 버튼을 누릅니다.

확정 방법은 두 가지입니다.

1. 봇이 계산한 진행 가능일 중 하나 이상 선택
2. `직접 입력으로 확정`을 눌러 날짜와 시간 직접 입력

스케줄을 확정하면 공대원의 추가 입력을 받지 않으며, 달력에는 `레이드 진행 예정일`이 표시됩니다.

확정 전에는 날짜와 내용을 꼭 확인해 주세요. 현재는 확정 취소와 재수정 기능을 지원하지 않습니다.

### 5. 스케줄 삭제하기

`/스케줄목록`에서 `삭제: 스케줄 이름` 버튼을 누릅니다.

삭제된 스케줄은 더 이상 입력할 수 없으며 활성 스케줄 목록에서 사라집니다.

## 자주 묻는 질문

### 내가 누른 날짜 선택 화면이 다른 사람에게도 보이나요?

아니요. 날짜 선택과 입력 완료 메시지는 본인에게만 보입니다. 입력 결과는 공용 달력에 반영됩니다.

### 시간을 잘못 입력했어요

같은 날짜 버튼을 다시 누르고 올바른 시간을 입력하면 됩니다.

### 참여할 수 없는 날짜는 어떻게 표시하나요?

별도로 표시할 필요 없이 그 날짜를 선택하지 않으면 됩니다. 이미 입력했다면 `삭제` 또는 `없음`을 입력하세요.

### 일정 입력 버튼이 사라졌어요

공대장이 스케줄을 확정했거나 삭제한 경우에는 더 이상 일정을 입력할 수 없습니다.

### 명령어나 버튼이 보이지 않아요

디스코드를 새로고침한 뒤 다시 확인해 주세요. 계속 보이지 않으면 서버 관리자에게 문의하세요.
